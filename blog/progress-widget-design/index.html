Starting Point
The initial progress widget — a basic prototype showing fetch progress for FHIR resources. Functional but not visually refined. This kicked off the exploration.
Prototype
Initial Progress Widget
Basic progress display. Showed page numbers and FHIR resource type names directly — functional for developers but not patient-friendly.
Round 1
Divergent Exploration
Six wildly different approaches to visualize progress for ~20 FHIR resource types with pagination. Each showed page numbers inside active cells and used abbreviations like "Obs", "Cnd", "Med".
Feedback: These all assume we know how many pages there'll be upfront — we don't! Need to pre-allocate all space and track progress without adding dots or resizing.
Variant A
Heatmap Grid
GitHub contribution chart style — 5×4 grid of cells with 2-3 letter abbreviations. Color intensity shows data volume. Compact but abbreviations are cryptic.
Variant B
Waterfall Rows
Chrome DevTools network panel style — horizontal bars per resource, segmented by page. High readability but assumes known page counts.
Variant C
Treemap Blocks
Rectangles sized by data volume, like a disk usage visualizer. Novel way to show data shape, but layout shifts as data arrives — violates the stability constraint.
Variant D
Radial Segments
Apple Watch activity ring style — donut with 20 arcs, thickness proportional to volume. Visually striking but hard to read individual resource status.
Variant E
Minimal Ticker
Ultra-minimal text display — segmented progress bar with a big resource count and cycling status text. Numbers are the hero element.
Variant F
Mosaic Chips
Tag cloud of inline text chips with colored backgrounds. Most readable names but medium density. Flows like text with category headers.
Round 2
Refined Data Model
New approach: exactly 44 fixed query slots grouped into 7 categories. Each cell represents one query, and page numbers appear inside active cells. Introduced a proper data spec.
Feedback: Still showing page numbers — patients don't care about "page 3". Also using FHIR jargon. Need patient-friendly labels and no technical details.
Variant A
Pixel Grid
Fixed 11×4 grid of 28px squares. Page numbers shown as centered digits inside active cells. Group labels sit above grid rows.
Variant B
Grouped Mini-Bars
Each of 7 groups gets a horizontal row with segments for each query. Like a stacked bar chart — Observations row has 17 thin segments.
Variant C
Waffle Chart
44-cell waffle chart with rounded squares. Cells arranged by group with thin separators. Page numbers in active cells, result counts in completed ones.
Variant D
Timeline Feed
Vertical scrolling event log, like CI build output. Completed queries appear as compact lines with colored dots. Active queries pulse at top.
Variant E
Ring Counter
iOS activity ring filling as queries complete. Large ticking number inside ("1,847 resources"). 44 arc segments, grey→green as each settles.
Variant F
Dense Mosaic v2
Refined chip approach with group headers. Tiny inline chips (22px tall, 10px font) flow like words. Pending chips are outline-only, done chips show counts.
Round 3
Proper State Model
Introduced a formal 5-state machine (pending → active → done/empty/error) with exact simulation data for Early, Mid, and Complete snapshots. Tighter visual spec with 4-tier green intensity.
Feedback: Getting closer. But still showing page numbers in some variants, and some use FHIR names like "Observation" instead of "Labs". Need to be completely patient-facing.
Variant A
Pixel Grid — Light
26×26px cells in grouped rows. Tiny 9px uppercase group labels. 5 clear visual states. Clean but still technical-looking.
Variant B
Segmented Bar + Detail
Single 300×16px horizontal bar divided into 44 segments with group gaps. Active detail text below. Very compact at ~190px tall.
Variant C
Dot Matrix Rows
7 rows of 10px dots. Hollow circles for pending, filled teal for active (pulsing), green intensity for done. Page numbers float above active dots.
Variant D
Counter Hero + Chips
Big "1,490" resource count dominates. Active queries shown as pulsing teal chips below. Completed queries accumulate as small green chips at bottom.
Variant E
Grouped Inline Indicators
Each group is a single text line with inline status characters (█░·) — like a terminal. Active query names listed at right. Dense but abstract.
Variant F
Phase Cards
Three stacked cards (Resources / References / Attachments). Only active phase expanded. Embedded grid of 38×20px rounded rectangles inside.
Round 4
Patient-Friendly Language
New brief: no page numbers, no FHIR jargon anywhere. Groups renamed to "Lab & Vitals", "Conditions", "Documents", "Services", "Medications", "Clinical", "Admin". All 44 slots pre-allocated from t=0.
Feedback: Much better! The "Counter Hero" concept keeps rising to the top — patients care about "1,490 resources found" more than individual query status. Need to nail the final form.
Variant A
Dot Matrix
7 labeled rows of 10px dots. Hollow=pending, pulsing teal=active, green fill=done (4-tier intensity), grey fill=empty, red=error. Patient-friendly group names.
Variant B
Segmented Bar + Activity
Grouped segmented bar on top, tiny group labels below, running activity list underneath showing friendly names like "Fetching lab results…"
Variant C
Counter Hero + Accumulating Chips
Big number centerpiece — "1,490 resources found". Active indicator lists current downloads by friendly name. Completed groups appear as summary chips below.
Variant D
Grouped Blocks + Running Counts
7 rows with mini-blocks per query and per-group running count on the right ("523", "35"). Shows data distribution across categories.
Variant E
Single Grid + Floating Label
44 squares in a continuous grid, groups separated by larger gaps. Floating label on hover/active. Abstract art quality — the data landscape emerges.
Variant F
Compact Phase Card
Single bordered card. Top zone: flowing grid of 44 tiny rounded rectangles with group gaps. Bottom zone: summary line with count and active names.
Round 5
Final Candidates
The definitive round. Strict brief: zero layout shift, zero jargon, zero page numbers. 5-state dots with 4-tier green intensity. All 44 elements fixed from t=0. Three phases (Resources → References → Attachments) with pre-allocated progress bars.
Decision: Variant D — "Counter Hero + Dot Strip" — selected as the winner. Big number gives emotional feedback, tiny dot strip shows the gestalt, pre-allocated phase bars prevent layout shift. Refined with pre-allocated References/Attachments bars.
Variant A
Dot Matrix Rows
7 rows, 10px dots, 4px gaps. 22px row height. Below: "1,490 resources found" + "32 of 44" + active query names in muted italic.
Variant B
Thin Segmented Bar + Counter
Three stacked zones: 16px segmented bar with 44 segments, counter ("1,490 resources found"), and 3-line activity feed with friendly names.
Variant C
Square Grid with Group Gaps
44 continuous 24px squares. Group gaps at 6px, within-group at 2px. High-count cells show number ("342"). Pulsing scale animation for active.
Variant D
Counter Hero + Dot Strip ★ CHOSEN
The winner. Large "1,490" resource count dominates. Single row of 44 tiny 7px dots below — greens accumulate, empties fade, errors pop. ~140px tall. Most minimal, most emotionally resonant.
Variant E
Stacked Group Bars
7 rows of segmented bars with per-group running counts on the right ("540", "35", "323"). Shows data distribution clearly but taller at ~230px.
Variant F
Fixed Chip Grid
All 44 chips with labels always visible ("Labs", "Vitals", "FamHx"). Most readable but tallest at ~350px. Active chips have teal borders, done turn green.

The Winner: Counter Hero + Dot Strip

After the initial selection, one refinement pass: pre-allocate the References and Attachments progress bars in all states so the layout never shifts. Then handed off for implementation.
